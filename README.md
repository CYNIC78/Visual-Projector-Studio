# VP Studio — Visual Projector Studio

Фреймворк для интерактивного ролеплея с локальными AI-моделями.  
Всё крутится на Neutralino.js — никаких Node.js, никаких Electron-жирняков. Просто `index.html`, пачка скриптов и стилей.

## Что это вообще

Это десктопное приложение, которое подключается к любой OpenAI-совместимой локальной модели (LM Studio, Ollama, что угодно) и даёт ей возможность показывать тебе картинки, переключать сцены, накладывать эффекты и играть в мини-игры. 

Работает так: ты общаешься с моделью, а она через теги типа `[IMG:forest]`, `[FX:lightning]`, `[TAB:open:characters]` управляет тем, что ты видишь на экране. Как текстовый квест, только с картинками.

## Быстрый старт

1. Скачай релизную сборку под свою платформу (Windows/Linux/Mac).
2. Запусти `neutralino-win_x64.exe` (или `neutralino-linux_x64`, `neutralino-mac_x64`).
3. Открой вкладку «Model» и впиши эндпоинт своей модели (например `http://127.0.0.1:1234/v1/chat/completions` для LM Studio).
4. Нажми «Load models» — выбери модель.
5. Кидай картинки в галерею (кнопки 📂 и 📎 или просто перетащи в окно).
6. Жми 🎬 Begin.

## Как оно устроено

- **visual-projector.js** — ядро: состояние, парсер тегов, переключение кадров, плеер, UI проектора.
- **projector-gallery.js** — управление галереей: ассеты, теги, табы, категории.
- **projector-session.js** — вся работа с моделью: отправка сообщений, стриминг, тулы, лог.
- **projector-chats.js** — управление несколькими чатами и профилями персонажей.
- **projector-games.js** — мини-игры и активности.
- **vp-subtitles.js** — субтитры поверх проектора.
- **vp-storage.js** — единое хранилище (IndexedDB + память + localStorage как запасной вариант).
- **vp-tools.js** — плагины для модели (броски кубиков, поиск по галерее и т.п.).
- **vp-core-tools.js** — встроенные тулы, которые идут сразу «из коробки».
- **fx-core.js** — движок визуальных эффектов (вспышки, туман, свечение и т.п.).
- **projector-shell.js** — рабочее пространство с перетаскиваемыми панелями.

Всё загружается в `index.html` в правильном порядке. Если что-то сломалось после изменений — в первую очередь проверь порядок скриптов.

## Команды, которые понимает модель

Модель может вставлять в ответ такие штуки:

| Команда | Что делает |
|---------|-----------|
| `[IMG:tag]` | Показывает картинку с этим тегом |
| `[IMG:tag:transition]` | То же, но с анимацией (crossfade, slide_left, zoom, pop, flip) |
| `[FX:name]` | Визуальный эффект (shockwave, lightning, fog, glow и т.п.) |
| `[FX:name:8]` | Эффект с интенсивностью от 1 до 10 |
| `[TAB:open:name]` | Открыть таб в галерее |
| `[TAB:collapse:name]` | Свернуть таб |
| `[CAT:open:name]` | Открыть категорию |
| `[CAT:collapse:name]` | Свернуть категорию |
| `[ACTIVITY_REQUEST]`, `[ACTIVITY_START]` и т.п. | Команды для мини-игр |
